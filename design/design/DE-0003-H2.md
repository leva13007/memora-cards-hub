## 1. Purpose
Define the visual and semantic rules for the `H2` typography atom so secondary headings are consistent across themes and breakpoints.

## 2. Scope
Applies to the `H2` UI-kit component only.
Does not define page-level heading hierarchy policies (usage rules live outside this doc).

## 3. Terminology
- Breakpoint: Mobile / Tablet / Desktop (see `design/foundations/breakpoints.md`)
- Theme: Light / Dark (see `design/foundations/theme.md`)

- Breakpoints: [design/foundations/breakpoints.md](../foundations/breakpoints.md)
- Theming: [design/foundations/theme.md](../foundations/theme.md)
- Tokens:

	- [design/foundations/tokens.md#colours](../foundations/tokens.md#colours)

## 5. Behavioral Model

### 5.1 Semantic element
- `H2` renders as a semantic heading level 2 (`<h2>`).

### 5.2 Typography rules
`H2` uses token-driven typography:
- Font family: `inherit` (must resolve to `Inter` via global typography stack)
- Line height: `normal`
- Font weight: `var(--font-weight-semi)`
- Color: `var(--color-text)` (theme-driven)
- Font size: `var(--font-size-h-2)` (breakpoint-driven)

### 5.3 Font size by breakpoint

| Token             | Mobile    | Tablet   | Desktop |
|-------------------|-----------|----------|---------|
| --font-size-h-2   | 1.125rem  | 1.25rem  | 1.5rem  |

## 6. Tokens & Variables

### Required tokens
- `--font-size-h-2`
- `--font-weight-semi`
- `--color-text`

## 7. States & Variants
- Themes: Light / Dark
- Breakpoints: Mobile / Tablet / Desktop
- No additional component variants are defined in this document.

## 8. Rules & Constraints
- Tickets must not define numeric typography values; they must reference token names and this design doc.
- `H2` must not hard-code font-size values; it must use `var(--font-size-h-2)`.
- `H2` must not apply custom line-height; it uses `line-height: normal`.
- Theme is a global concern:
	- `H2` must rely on semantic tokens (e.g. `--color-text`) and must not reference theme-specific values directly.
	- `H2` must not manage or override the active theme locally (see `design/foundations/theme.md`).

## 9. Accessibility
- The component must use a semantic `<h2>` element (heading level 2).
- Color must remain readable in both themes.
	- Theme-dependent values are provided by design tokens scoped under the global theme activation model (see `design/foundations/theme.md`).

## 10. Non-normative Implementation Notes
For clarity only (tokens are the source of truth):

```css
h2 {
	color: var(--color-text);
	font-family: inherit;
	font-size: var(--font-size-h-2);
	font-weight: var(--font-weight-semi);
	line-height: normal;
}
```

## 11. Change Impact
- Any token changes to `--font-size-h-2`, `--font-weight-semi`, or `--color-text` require re-running H2-related test runs.
- Changes may affect section titles and secondary headings across the product.

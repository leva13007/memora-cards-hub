## 1. Purpose
Define the visual and semantic rules for the `H1` typography atom so primary headings are consistent across themes and breakpoints.

## 2. Scope
Applies to the `H1` UI-kit component only.
Does not define page-level heading hierarchy policies (usage rules live outside this doc).

## 3. Terminology
- Breakpoint: Mobile / Tablet / Desktop (see `design/foundations/breakpoints.md`)
- Theme: Light / Dark

## 4. Design Dependencies
- Breakpoints: [design/foundations/breakpoints.md](../foundations/breakpoints.md)
- Theming: [design/foundations/theme.md](../foundations/theme.md)
- Tokens: [design/foundations/tokens.md#typography](../foundations/tokens.md#typography)

## 5. Behavioral Model

### 5.1 Semantic element
- `H1` renders as a semantic heading level 1 (`<h1>`).

### 5.2 Typography rules
`H1` uses token-driven typography:
- Font family: `inherit` (must resolve to `Inter` via global typography stack)
- Line height: `normal`
- Font weight: `var(--font-weight-bold)`
- Color: `var(--color-text)` (theme-driven)
- Font size: `var(--font-size-h-1)` (breakpoint-driven)

### 5.3 Font size by breakpoint

| Token             | Mobile   | Tablet    | Desktop   |
|-------------------|----------|-----------|-----------|
| --font-size-h-1   | 1.5rem   | 1.75rem   | 2rem      |

## 6. Tokens & Variables

### Required tokens
- `--font-size-h-1`
- `--font-weight-bold`
- `--color-text`

## 7. States & Variants
- Themes: Light / Dark
- Breakpoints: Mobile / Tablet / Desktop
- No additional component variants are defined in this document.

## 8. Rules & Constraints
- Tickets must not define numeric typography values; they must reference token names and this design doc.
- `H1` must not hard-code font-size values; it must use `var(--font-size-h-1)`.
- `H1` must not apply custom line-height; it uses `line-height: normal`.
- Theme is a global concern:
	- `H1` must rely on semantic tokens (e.g. `--color-text`) and must not reference theme-specific values directly.
	- `H1` must not manage or override the active theme locally (see `design/foundations/theme.md`).

## 9. Accessibility
- The component must use a semantic `<h1>` element (heading level 1).
- Color must remain readable in both themes.
	- Theme-dependent values are provided by design tokens scoped under the global theme activation model (see `design/foundations/theme.md`).

## 10. Non-normative Implementation Notes
For clarity only (tokens are the source of truth):

```css
h1 {
	color: var(--color-text);
	font-family: inherit;
	font-size: var(--font-size-h-1);
	font-weight: var(--font-weight-bold);
	line-height: normal;
}
```

## 11. Change Impact
- Any token changes to `--font-size-h-1`, `--font-weight-bold`, or `--color-text` require re-running H1-related test runs.
- Changes may affect page titles and primary headings across the product.

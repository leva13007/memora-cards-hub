# TR-MC-0001 — Sprint 01: UI-kit compnents verification — Storybook manual run

## Purpose
Run a focused manual verification of the UI-kit components created in Sprint 01:
- `TextLight`, `TextMedium`, `TextBold` (MC-0001)
- `TextLink` (MC-0002)
- `H1` (MC-0003)
- `H2` (MC-0004)
- `Container` (MC-0005)
- `Breadcrumbs` (MC-0006)
Validate responsive behavior, theme colors, core props, and accessibility compliance.

## Scope
### In Scope
- All components listed above per their respective acceptance criteria.
- Breakpoints:
	- Desktop
	- Tablet
	- Mobile
- Themes:
	- Light
	- Dark

### Out of Scope
- Cross-browser matrix (Safari/Firefox/etc.)
- Full app-level integration (only UI-kit components / Storybook)
- Accessibility beyond basic semantic rendering.
- Performance profiling

## Preconditions
- Storybook is available and includes stories for all components listed above.
- Theme switcher is available (or Storybook provides theme toggle via toolbar).
- Browser devtools are available to inspect computed styles and tokens.
